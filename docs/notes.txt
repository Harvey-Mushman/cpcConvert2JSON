CFMiX CPC Batch Converter - Notes
===================================

WHAT THIS DOES
--------------
cpcBatchConverter.py reads PDF certificates from the ./certificates folder,
sends each one to the Claude AI API, and saves the extracted data as JSON
files in the ./json_output folder.


BEFORE RUNNING THE SCRIPT
--------------------------
You need an Anthropic API key. Get one at: https://console.anthropic.com/

1. THe file anthropicKey.txt contains the CPC_Convert2JSON key which is not backed up on git.
2. Run the script:

    python cpcConvert.py
    python cpcMerge.py

NOTE: anthropicKey.txt is listed in .gitignore so it will NOT be uploaded to GitHub.

Activate Python Enviroment
(venv) PS C:\Users\hm\Documents\CFMiX\cpcConvert2json>

Requirements:

pip install anthropic
pip install pdfplumber


FOLDER STRUCTURE
----------------
cpcConvert2json/
  cpcConvert.py   <- the script
  cpcMerge.py     <- combines all of hte page_# files into the json_output file
  cpcCount.py     <- ounts the commodities in each individual page_*.json file and shows a per-page breakdown.
  cpcPdfCheck.py  <- scans each page of a certificate PDF and reports whether it's: text-searchable, scanned image or both text and scanned.
  notes.txt       <- this file
  certificates/   <- put your PDF files here (create if missing)
  json_output/    <- JSON results will appear here (auto-created)
  page_output/    <- page.001 through the page-### that was converted one page at a time

THe import from Ventura was created from their Access system and exoported as XLS filetype.  Here is the format:

VENTURA_ACCESS_EXPORT.json — config for .xls Access database exports (positional column mapping)
VENTURA_51-049_Rev_10-24.json — config for .xlsx state form (section-based with labeled rows)

layout:

Col	Field
0	issuing_county
1	certificate_number
2	(internal ID — skip)
3	producer name (contact)
4	issuing_date (Excel serial)
5	farm_name
6	expiration_date (Excel serial)
7-8	address (col 8 is overflow)
9	city
10	zip_code
11	county_fee
12	phone_business
13	phone_cell
14-18	production_sites (up to 5, "N/A" = empty)
19	(unused — always N/A)
20	storage_location
21	authorized_counties_count (?)
22	additional storage location
Commodities in columns 24-29: row_num, crop, acres, season, volume, location_code

(VENTURA_51-049_Rev_10-24.json) for the XLSX form layout (like Shigeru)
Build extract_xls_access_export() — reads the flat row-0 + commodity rows
Build extract_xlsx_form_based()
